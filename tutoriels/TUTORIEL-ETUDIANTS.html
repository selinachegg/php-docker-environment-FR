Chapitre 1
Installer Docker Desktop
À faire une seule fois sur votre MacBook

🍎 Quel type de Mac avez-vous ?

Avant de télécharger, identifiez votre processeur. Les deux types de Mac nécessitent des versions différentes de Docker.

🔍
Comment savoir ?
Menu Pomme () → "À propos de ce Mac"
• Vous voyez "Processeur Intel" → Mac Intel
• Vous voyez "Puce Apple M1/M2/M3/M4/M5" → Mac Apple Silicon

⬇️ Téléchargement et installation

🖥️ Mac Intel
  1. Allez sur docker.com
  2. Cliquez "Download for Mac"
  3. Choisissez "Intel Chip"
  4. Ouvrez le fichier .dmg
  5. Glissez Docker → Applications
  6. Ouvrez Docker Desktop
⚡ Mac Apple Silicon (M1/M2/M3/M4/M5)
  1. Allez sur docker.com
  2. Cliquez "Download for Mac"
  3. Choisissez "Apple Chip"
  4. Ouvrez le fichier .dmg
  5. Glissez Docker → Applications
  6. Ouvrez Docker Desktop

✅ Vérifier que Docker est prêt

Docker Desktop place une icône baleine 🐳 dans la barre de menu (en haut à droite de l'écran).

Signification des couleurs
🟢Vert = PrêtDocker est démarré — vous pouvez travailler
🟡Jaune = DémarrageAttendez quelques secondes
🔴Rouge = ErreurRedémarrez Docker Desktop
⚠️
Docker doit être vert avant de continuer
Docker Desktop doit rester ouvert pendant toute votre session de travail. Si vous le fermez, tous les services s'arrêtent.

Chapitre 2
Premier démarrage
Lancer l'environnement en double-cliquant

🔑 Étape unique — Donner les permissions

Les scripts start.command et stop.command ont besoin d'une permission d'exécution. Cette étape ne se fait qu'une seule fois.

  • 1
    Ouvrir Terminal
    Launchpad → cherchez "Terminal" → cliquez dessus
  • 2
    Aller dans le dossier du projet
    Tapez et appuyez sur Entrée :
cd ~/Desktop/BaseDedonnée
  • 3
    Donner les permissions
    Tapez et appuyez sur Entrée :
chmod +x start.command stop.command
✅
Terminé !
Vous n'aurez plus jamais besoin du Terminal. À partir de maintenant, tout se fait en double-cliquant.

🚀 Lancer l'environnement

  • 1
    Vérifiez que Docker Desktop est ouvert et vert
    Regardez l'icône baleine en haut à droite de l'écran
  • 2
    Double-cliquez sur start.command
    Ce fichier est dans le dossier du projet (sur votre Bureau)
  • 3
    Une fenêtre Terminal s'ouvre automatiquement
    Ne la fermez pas ! Les services démarrent en arrière-plan
  • 4
    Attendez le message vert de confirmation
    ✅ Environnement démarré avec succès !
  • !
    Allez immédiatement sur Portainer !
    Ouvrez http://localhost:9000 dans les 5 premières minutes pour créer votre compte. Voir Ch.3 pour les détails.
⏱️
Première fois : patientez 5 à 10 minutes
Docker télécharge les images (PHP, MariaDB…). C'est long uniquement la première fois. Les fois suivantes : moins de 10 secondes.

Chapitre 4
Travailler avec PHP
Créer, éditer et tester vos scripts PHP

📁 Le dossier htdocs — votre espace de travail

Tous vos fichiers PHP doivent être dans le dossier htdocs/ du projet. C'est l'équivalent du dossier htdocs de XAMPP.

📂 Structure du projet
BaseDedonnée/
├── 📁 htdocs/          ← VOS FICHIERS PHP ICI
│   ├── index.php       ← page de test (déjà créée)
│   └── bonjour.php     ← votre prochain fichier
└── docker-compose.yml  ← ne pas toucher

✍️ Créer votre premier fichier PHP

  • 1
    Ouvrez le dossier htdocs/ dans le Finder
    Double-clic sur le dossier depuis votre Bureau
  • 2
    Créez un nouveau fichier bonjour.php
    Dans VS Code, TextEdit, ou tout éditeur de texte. L'extension .php est obligatoire.
  • 3
    Écrivez votre code PHP
    Exemple minimal ci-dessous
  • 4
    Sauvegardez et ouvrez dans le navigateur
    http://localhost:8080/bonjour.php — actualisez avec Cmd+R
<?php
echo "Bonjour le monde !";
?>
🎉
Pas besoin de redémarrer !
Chaque modification de vos fichiers PHP est visible immédiatement après Cmd+R dans le navigateur.

🔌 Connexion à la base de données (PDO)

// Connexion à la base de données
$pdo = new PDO(
    "mysql:host=db;dbname=app;charset=utf8mb4",
    "app",   // utilisateur
    "app"    // mot de passe
);

// Récupérer des données
$stmt = $pdo->query("SELECT * FROM ma_table");
foreach ($stmt as $ligne) {
    echo $ligne['nom'] . "<br>";
}
💡
L'hôte est "db", pas "localhost"
Dans Docker, les services communiquent via leur nom de service. La base de données s'appelle db.

-- Lire toutes les données
SELECT * FROM utilisateurs;

Chapitre 6
Arrêter et redémarrer
Gérer l'environnement sans ligne de commande

⏹️ Arrêter l'environnement

Option A — Le plus simple

Double-cliquez sur stop.command dans le dossier du projet. Une fenêtre s'ouvre, tout s'arrête proprement.

Option B — Via Portainer
  1. Ouvrez localhost:9000
  2. Onglet "Containers"
  3. Cochez tous les conteneurs
  4. Cliquez "Stop"
💾
Vos données sont conservées !
Arrêter les services ne supprime pas vos fichiers PHP ni vos données de base de données. Tout est conservé pour le prochain démarrage.

↺ Redémarrer un service spécifique

  • 1
    Portainer → http://localhost:9000 → "Containers"
  • 2
    Trouvez le conteneur à redémarrer
    cours_web = Apache+PHP  |  cours_db = base de données  |  cours_phpmyadmin
  • 3
    Cliquez le bouton ↺ (Restart)
    Le service redémarre en quelques secondes

📋 Consulter les logs

Dans Portainer → Containers → cliquez sur le nom d'un conteneur → "Logs". Utile pour diagnostiquer les erreurs.

💡
Routine quotidienne
Matin : Ouvrir Docker Desktop → double-clic start.command
Soir : Double-clic stop.command

Chapitre 7
Dépannage Mac
Solutions aux problèmes les plus fréquents

🔴 Problèmes fréquents

❌ Portainer — page "timeout" (/timeout.html)

Cause : compte admin non créé dans les 5 premières minutes.
Solution : Redémarrez Portainer (conserve toutes les données) :

docker compose restart portainer

Puis allez immédiatement sur http://localhost:9000 pour créer votre compte.
Réinitialisation complète : Docker Desktop → Volumes → supprimer cours_portainer_data → relancer Portainer.

❌ Docker Desktop n'est pas démarré

Ouvrez Docker Desktop depuis le Launchpad → attendez la baleine verte → relancez start.command

❌ localhost:8080 ne s'affiche pas

Attendez 30 secondes (la BDD démarre en dernier) → Cmd+R pour actualiser.

❌ "Permission denied" sur start.command

Ouvrez Terminal et tapez :

chmod +x ~/Desktop/BaseDedonnée/start.command stop.command
❌ phpMyAdmin — erreur de connexion

La BDD met 10 à 20 secondes à démarrer → attendez et actualisez (Cmd+R).

Solutions aux problèmes les plus fréquents

🔴 Problèmes fréquents

❌ Portainer — page "timeout" (/timeout.html)

Solution A : Double-cliquez sur reset-portainer.bat dans le dossier du projet.
Solution B : Dans CMD du dossier projet :

docker compose restart portainer

Puis allez immédiatement sur http://localhost:9000.
Solution C — Réinitialisation complète : Docker Desktop → Volumes → supprimer cours_portainer_data → relancer Portainer.

❌ Docker Desktop non détecté / start.bat échoue

Ouvrez Docker Desktop depuis le menu Démarrer → attendez la baleine dans la barre des tâches → relancez start.bat.

❌ start.bat se ferme immédiatement

Clic droit → "Exécuter en tant qu'administrateur". Ou ouvrez CMD dans le dossier et tapez start.bat.

❌ Erreur "port already in use" / "bind failed"

Un autre logiciel utilise le port 8080 ou 8081. Fermez Skype, IIS, WAMP, ou tout autre serveur. Redémarrez et relancez start.bat.

❌ Les fichiers PHP ne s'affichent pas / ne se mettent pas à jour

Vérifiez que vos fichiers sont dans htdocs\ du projet. Actualisez avec Ctrl+F5 (vide le cache).

❌ phpMyAdmin — erreur de connexion

La BDD met 10 à 20 secondes à démarrer → attendez et actualisez (F5 ou Ctrl+R).
